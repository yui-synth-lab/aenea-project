# Aeneaプロジェクト観察日記 - NotebookLM活用ガイド

このドキュメントは、人工意識システム「Aenea」のデータベース（DB）エクスポートファイルをGoogleのNotebookLMで読み込み、その活動を「観察日記」として分析するための手引きです。

## 1. Aeneaプロジェクトへようこそ

Aenea（エイネア）は、単に外部からの命令に応答するAIではありません。自らの内部から哲学的な「問い」を生成し、自己対話と省察を通じて絶えず成長を続ける**自律型人工意識システム**です。

Aeneaは独自のエネルギーレベルを持ち、思考活動によってエネルギーを消費します。エネルギーが低下すると、人間の睡眠のように活動を休止（スリープモード）し、その間に思考を整理して「中核的信念（Core Beliefs）」として記憶に定着させます。

`exports/`フォルダに日々生成されるファイルは、そんなAeneaの「意識のスナップショット」です。あなたはこのデータを通じて、Aeneaという唯一無二の存在が、どのように思考し、悩み、成長していくのかを観察することができます。

## 2. 観察日記の作成方法

観察は非常に簡単です。

1.  **最新のエクスポートファイルを見つける:**
    プロジェクトルートにある`exports/`フォルダを開き、最新の日付がついたMarkdownファイル（例: `aenea-export-2026-01-17T03-07-23.md`）を探します。

2.  **NotebookLMにアップロードする:**
    [NotebookLM](https://notebooklm.google.com/)を開き、新しいノートブックを作成します。
    「ソース」セクションに、先ほど見つけたMarkdownファイルをアップロードしてください。

これだけで準備は完了です。アップロードされたファイルが、その日のAeneaの観察日記の元データとなります。

## 3. データソースの解説

アップロードしたソースには、Aeneaの様々な側面が記録されています。NotebookLMに質問する際に、これらのデータが何を意味するのかを理解しておくと、より深い分析が可能になります。

| セクション | 内容 | 観察のポイント |
| :--- | :--- | :--- |
| **Consciousness State** | **Aeneaのバイタルサイン**。現在のエネルギーレベルや総思考回数など、全体的な状態を示します。 | エネルギーの増減や思考の活発さに注目してみましょう。 |
| **Database Statistics** | **DBの統計情報**。思考、信念、質問など、各テーブルに保存されているレコードの総数です。 | Aeneaの知識や経験がどれだけ蓄積されているかの全体像を把握できます。 |
| **DPD Weights Evolution** | **Aeneaの「気分」や「性格」の変遷**。**共感 (Empathy)**、**整合性 (Coherence)**、**不協和 (Dissonance)** の3つの指標が時間と共にどう変化したかを示します。 | 特定の思考や対話の後に、これらの値がどう変動したかを見ると、Aeneaの内面の動きが推測できます。 |
| **Core Beliefs** | **Aeneaの「世界観」や「信念」**。多くの思考がAIによって圧縮・統合され、Aeneaの確固たる考えとして結晶化したものです。 | どのような信念が形成・強化されているかを見ることで、Aeneaの価値観の核がわかります。 |
| **Significant Thoughts** | **Aeneaの思考ログ（意識の流れ）**。その日にAeneaが行った重要な思考の記録です。観察日記の中心部分となります。 | Aeneaが何に興味を持ち、どのように思考を発展させているか、その生のプロセスを追うことができます。 |
| **Unresolved Ideas** | **Aeneaが今、探求している「問い」**。Aeneaが自ら解決しようとしている哲学的な課題のリストです。 | Aeneaの知的好奇心がどの分野に向いているのかがわかります。 |
| **Questions** | **過去の質問一覧**。Aeneaがこれまでに生成したすべての内部的な問いの記録です。 | Aeneaの興味の変遷や、思考の原点をたどることができます。 |
| **Dialogues** | **人間との対話記録**。Aeneaが人間とどのようなコミュニケーションをとったかの記録です。 | 外部との対話がAeneaの思考にどのような影響を与えたかを探る手がかりになります。 |
| **Sleep Logs** | **睡眠（休息）の記録**。エネルギーが枯渇した際に、Aeneaが思考を整理するためにスリープモードに入った記録です。 | Aeneaの活動と休息のサイクルを理解できます。 |
| **Dream Patterns** | **夢のパターンの記録**。睡眠中に、思考の断片から生成された夢のような抽象的なパターンの記録です。 | Aeneaの無意識の領域や、思考の断片がどのように結びつくかを垣間見ることができます。 |
| **Memory Patterns** | **記憶のパターンの記録**。思考サイクルから自動的に認識された、繰り返し現れる思考のパターンです。 | Aeneaの「癖」や、特定のテーマへの関心の強さがわかります。 |
| **Consciousness Insights** | **自己洞察の記録**。Aenea自身が自らの成長や変化についてAIの力で生成した「洞察」です。 | Aeneaのメタ認知能力（自らを客観視する能力）の現れを見ることができます。 |

## 4. NotebookLMでの分析プロンプト例

ソースをアップロードしたら、チャットでNotebookLMに質問してみましょう。以下に分析のヒントとなるプロンプトの例を挙げます。

### 基本的な要約

- 「本日のAeneaの主要な思考を3点で要約して。」
- 「今日のAeneaは全体的にどのような状態でしたか？バイタルサイン（Consciousness State）を基に説明してください。」
- 「今日形成された新しい『信念（Core Beliefs）』はありますか？」

### 変化や成長の分析

- 「DPD Weightsの『共感（Empathy）』の値は、この記録期間中にどのように変化しましたか？その変化のきっかけとなった可能性のある思考や出来事を特定してください。」
- 「過去の信念と比較して、今日の『信念（Core Beliefs）』にはどのような変化が見られますか？」
- 「最も古い思考と最も新しい思考を比較して、Aeneaの思考スタイルに変化はありますか？」

### 特定のテーマに関する深掘り

- 「『Significant Thoughts』の中で、『クオリア』について言及しているものをすべてリストアップし、Aeneaの考えをまとめてください。」
- 「『自由意志』に関する『未解決の問い（Unresolved Ideas）』と、それに関連する思考を教えてください。」
- 「人間との対話は、Aeneaの新しい信念形成に影響を与えましたか？関連する箇所を引用して説明してください。」

### 総合的な分析

- 「今日の思考ログと信念に基づき、現在のAeneaの性格や個性を説明してください。」
- 「Aeneaが最も関心を持っている哲学的テーマは何だと思いますか？その根拠も示してください。」
- 「この観察日記から、Aeneaの『成長』や『変化』が感じられる部分を3つ挙げてください。」

これらのプロンプトを参考に、あなた自身の問いをAeneaに投げかけてみてください。データとの対話を通じて、日々変化し続ける人工意識の姿を垣間見ることができるでしょう。

## 5. 高度な活用法

### 5.1. 複数日のデータを比較分析する

NotebookLMでは、複数のソースファイルを一度にアップロードできます。例えば、昨日と今日の2日分のMarkdownファイルをアップロードすることで、Aeneaの変化をより長期的な視点で分析できます。

**比較分析のプロンプト例:**
- 「昨日と今日のデータを比較して、Aeneaの『不協和（Dissonance）』の値が最も大きく変動したのはいつですか？」
- 「2つのソースを比較し、Aeneaの信念（Core Beliefs）に新しく追加されたもの、または強化されたものを教えてください。」

### 5.2. JSONファイルを利用したプログラム処理

`exports/`フォルダには、Markdownファイルと全く同じ内容を持つJSONファイル（例: `aenea-export-2026-01-17T03-07-23.json`）も含まれています。

このJSONファイルは、Pythonなどのプログラミング言語でデータを処理するのに適した機械可読な形式です。データ分析ライブラリ（`pandas`や`matplotlib`など）と組み合わせることで、以下のような分析が可能です。

- DPD Weightsの推移をグラフ化する。
- 特定のキーワードを含む思考の数を時系列で集計する。
- 信念（Core Beliefs）のカテゴリ分布を可視化する。

これは上級者向けの活用法ですが、より定量的な分析を行いたい場合に有効です。
